Create LED Blink Update

A simple script to prove OTA works

GPIO Wiring

Pin 8 (GPIO14) → LED (+) → LED → 220Ω-2.2kΩ → Pin 6 (GND)

Create Update Script

mkdir -p ~/beagley-ota-project/ota-files cd ~/beagley-ota-project cat > ota-files/blink.sh << 'EOF' #!/bin/bash echo "OTA Update Success! LED blinking..." while true; do gpioset $(gpiofind GPIO14)=1 # ON sleep 0.5 gpioset $(gpiofind GPIO14)=0 # OFF sleep 0.5 done EOF chmod +x ota-files/blink.sh

Create Your Update: LED Blink Script

Visual proof that OTA works - an LED that blinks via deployed code

Understanding BeagleY-AI GPIO

# BeagleY-AI uses Linux GPIO character device interface (libgpiod) # 40-pin header follows Raspberry Pi layout for compatibility # GPIO14 = Physical Pin 8 (4th pin down on inner row) # Wiring diagram: Pin 8 (GPIO14) ──>[LED Anode (+)]──>[LED]──>[220Ω-2.2kΩ]──> Pin 6 (GND) # Long leg Red/Green/Yellow Any ground pin

Create the Update Script

# On your workstation - create update directory mkdir -p ~/beagley-ota-project/ota-files cd ~/beagley-ota-project # Create the blink script with proper error handling cat > ota-files/blink.sh << 'EOF' #!/bin/bash # BeagleY-AI GPIO LED Blink - Deployed via Thistle OTA # Demonstrates successful cryptographically-verified update # GPIO Reference: https://docs.beagleboard.org/boards/beagley/ai/demos/using-gpio.html set -euo pipefail # Exit on error, undefined vars, pipe failures echo "======================================" echo "🎉 OTA Update Successfully Deployed!" echo "======================================" echo "Version: 1.0.0" echo "Target: GPIO14 (Physical Pin 8)" echo "Press Ctrl+C to stop" echo "" # Find the GPIO chip and line for GPIO14 # gpiofind returns: gpiochip0 14 (chip and line number) if ! GPIO_INFO=$(gpiofind GPIO14 2>/dev/null); then echo "ERROR: Cannot find GPIO14. Is libgpiod installed?" echo "Install with: sudo apt-get install gpiod" exit 1 fi GPIO_CHIP=$(echo $GPIO_INFO | cut -d' ' -f1) GPIO_LINE=$(echo $GPIO_INFO | cut -d' ' -f2) echo "Using: $GPIO_CHIP line $GPIO_LINE" echo "Starting blink pattern..." # Trap Ctrl+C to clean up GPIO state trap "echo -e '\nCleaning up...'; gpioset $GPIO_CHIP $GPIO_LINE=0; exit 0" SIGINT SIGTERM # Main blink loop - 0.5 second on/off for visibility while true; do gpioset $GPIO_CHIP $GPIO_LINE=1 # LED ON - High (3.3V) sleep 0.5 gpioset $GPIO_CHIP $GPIO_LINE=0 # LED OFF - Low (0V) sleep 0.5 done EOF # Make executable chmod +x ota-files/blink.sh # Test locally if you have gpioset installed head -n 5 ota-files/blink.sh
Why a blink script? It's simple, visual, and proves the entire OTA chain works - from signing to deployment to execution.

Quick Troubleshooting

Common issues and instant fixes

Can't SSH?

  • Wait 90 seconds after boot
  • Try: 192.168.7.2 (USB)
  • Check: beagley-ota.local
  • Verify USB cable has data

Update Not Fetching?

  • Check: ping google.com
  • Verify THISTLE_TOKEN set
  • Run: tuc -v (verbose mode)
  • Check Thistle dashboard

LED Not Blinking?

  • GPIO14 = Physical Pin 8
  • LED long leg = positive
  • Need 2.2kΩ resistor
  • Test: gpioset 0 14=1
